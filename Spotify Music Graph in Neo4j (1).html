Recommended artist normalization step
# Normalize the artists field before splitting
# If your raw file already uses semicolons, this is simple.
# If your raw file uses another separator, convert it to semicolons first.

df["artists"] = df["artists"].astype(str).str.strip()

def normalize_artists(value):
    parts = [p.strip() for p in value.split(";")]
    parts = [p for p in parts if p]   # remove blanks
    return ";".join(parts)

df["artists"] = df["artists"].apply(normalize_artists)

Accepted simplification for this course project

The raw Spotify dataset can contain the same track under more than one genre. In this assignment, keeping one row per track_id and therefore one chosen genre per track is acceptable, as long as you use that same cleaned track base consistently for all CSV files.

Optional but recommended first step

Create uniqueness constraints before you import. This helps Neo4j match nodes more efficiently and helps prevent accidental duplicates. If your Neo4j version does not support the exact optional constraint syntax shown here, you may skip this optional step and continue with the imports.

Optional but recommended constraints before import
CREATE CONSTRAINT track_id_unique IF NOT EXISTS
FOR (t:Track) REQUIRE t.track_id IS UNIQUE;

CREATE CONSTRAINT artist_name_unique IF NOT EXISTS
FOR (a:Artist) REQUIRE a.name IS UNIQUE;

CREATE CONSTRAINT album_name_unique IF NOT EXISTS
FOR (al:Album) REQUIRE al.name IS UNIQUE;

CREATE CONSTRAINT genre_name_unique IF NOT EXISTS
FOR (g:Genre) REQUIRE g.name IS UNIQUE;

Create Track nodes

WITH row

Part 4

Cypher query tasks with exact clarifications

For each task, write the Cypher query, make sure it works on your data, and measure the execution time.

Task 1

Highly popular tracks

  • Return the top 20 tracks with popularity > 80.
  • Return track_name and popularity.
  • Sort by popularity descending and use LIMIT 20.
Task 2

Tracks with “Love” or “Dream” in the title

  • Find up to 50 tracks whose title contains love or dream, case-insensitive.
  • Using toLower(t.track_name) is a good approach.
  • Return track_name and popularity, sorted by popularity descending.
Task 3

Tracks by a specific artist

  • Choose an artist name that exists in your data. Taylor Swift is only an example.
  • Use the PERFORMED_BY relationship.
  • Return track_name and popularity, sorted by popularity descending.
Task 4

Upbeat and happy tracks

  • Choose your thresholds, for example danceability > 0.7 and valence > 0.7.
  • Also require popularity > 60.
  • Return track name, popularity, danceability, and valence.
  • In your written explanation, clearly state the thresholds you used.
Task 5

Update artist name

  • Rename artist Jason Mraz to Jason M. if that artist exists in your data.
  • If Jason Mraz is not present, choose another existing artist and clearly state which one you renamed.
  • Then query the updated artist name to confirm the change.
Reset before each timed run
Task 6

Average track popularity by genre

  • Use HAS_GENRE.
  • Return genre name and average popularity.
  • Sort by average popularity descending.
Task 7

Genre-wise track counts

  • Use HAS_GENRE.

Execution time

How to measure time correctly

Run every task 10 times and compute the average execution time.

Timer to usetime.perf_counter() is recommended.
Neo4j driverYou may use the Neo4j Python driver with GraphDatabase.driver(...).
Reset ruleOnly Task 5 changes the graph, so reset the artist name before each timed run of that task.
Important for the update task

If you rename the same artist 10 times without resetting the original name, the later runs are not timing the same update. Reset the artist to the original name before each new timed run.

Python timing pattern
import time
from neo4j import GraphDatabase

driver = GraphDatabase.driver(
    "bolt://localhost:7687",
    auth=("neo4j", "password")
)

# Use "spotifydb" if your environment supports a separate database for this assignment.
# If your setup only gives you the default database, replace DB_NAME with that database name
# (often "neo4j"), or omit database=DB_NAME and use the default session.
DB_NAME = "spotifydb"

query = '''
MATCH (t:Track)
WHERE t.popularity > 80
RETURN t.track_name AS track_name, t.popularity AS popularity
ORDER BY t.popularity DESC
LIMIT 20
'''

with driver.session(database=DB_NAME) as session:
    times = []
    for i in range(10):
        start = time.perf_counter()
        result = session.run(query).data()
        end = time.perf_counter()

        run_time = end - start
        times.append(run_time)
        print(f"Run {i+1}: {run_time:.6f} seconds")

    average_time = sum(times) / len(times)
    print("Average execution time (s):", average_time)

driver.close()
